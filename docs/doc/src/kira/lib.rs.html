2# Kira
3
4Kira is a backend-agnostic library to create expressive audio for games. It provides
5tweens for smoothly adjusting properties of sounds, a flexible [mixer](track)
6for applying [effects](effect) to audio, a [clock] system for precisely timing audio events,
7and [spatial audio](crate::track#spatial-tracks) support.
8
9To get started, create an [`AudioManager`] and use it to [play](AudioManager::play) a
10[`StaticSoundData`](crate::sound::static_sound::StaticSoundData) or
11[`StreamingSoundData`](crate::sound::streaming::StreamingSoundData).
12
13## Examples
14
15Playing a sound multiple times simultaneously:
16
17```rust, no_run
18# extern crate kira;
19#
20use kira::{
21	AudioManager, AudioManagerSettings, DefaultBackend,
22	sound::static_sound::{StaticSoundData, StaticSoundSettings},
23};
24
25// Create an audio manager. This plays sounds and manages resources.
26let mut manager = AudioManager::<DefaultBackend>::new(AudioManagerSettings::default())?;
27let sound_data = StaticSoundData::from_file("sound.ogg")?;
28manager.play(sound_data.clone())?;
29// After a couple seconds...
30manager.play(sound_data.clone())?;
31// Cloning the sound data will not use any extra memory.
32# Result::<(), Box<dyn std::error::Error>>::Ok(())
33```
34
35Gradually speeding up a sound over time:
36
37```rust, no_run
38# extern crate kira;
39#
40use std::time::Duration;
41
42use kira::{
43	AudioManager, AudioManagerSettings, DefaultBackend,
44	sound::static_sound::{StaticSoundData, StaticSoundSettings},
45	Tween,
46};
47
48let mut manager = AudioManager::<DefaultBackend>::new(AudioManagerSettings::default())?;
49let sound_data = StaticSoundData::from_file("sound.ogg")?;
50let mut sound = manager.play(sound_data)?;
51// Start smoothly adjusting the playback rate parameter.
52sound.set_playback_rate(
53	2.0,
54	Tween {
55		duration: Duration::from_secs(3),
56		..Default::default()
57	},
58);
59# Result::<(), Box<dyn std::error::Error>>::Ok(())
60```
61
62Playing a sound with a low-pass filter applied (this makes the
63audio sound muffled):
64
65```rust, no_run
66# extern crate kira;
67#
68use kira::{
69	AudioManager, AudioManagerSettings, DefaultBackend,
70	sound::static_sound::{StaticSoundData, StaticSoundSettings},
71	track::TrackBuilder,
72	effect::filter::FilterBuilder,
73};
74
75let mut manager = AudioManager::<DefaultBackend>::new(AudioManagerSettings::default())?;
76// Create a mixer sub-track with a filter.
77let mut track = manager.add_sub_track({
78	let mut builder = TrackBuilder::new();
79	builder.add_effect(FilterBuilder::new().cutoff(1000.0));
80	builder
81})?;
82// Play the sound on the track.
83let sound_data = StaticSoundData::from_file("sound.ogg")?;
84track.play(sound_data)?;
85# Result::<(), Box<dyn std::error::Error>>::Ok(())
86```
87
88Playing sounds in time with a musical beat:
89
90```rust, no_run
91# extern crate kira;
92#
93use kira::{
94	AudioManager, AudioManagerSettings, DefaultBackend,
95	sound::static_sound::{StaticSoundData, StaticSoundSettings},
96	clock::ClockSpeed,
97};
98
99const TEMPO: f64 = 120.0;
100
101let mut manager = AudioManager::<DefaultBackend>::new(AudioManagerSettings::default())?;
102// Create a clock that ticks 120 times per second. In this case,
103// each tick is one musical beat. We can use a tick to represent any
104// arbitrary amount of time.
105let mut clock = manager.add_clock(ClockSpeed::TicksPerMinute(TEMPO))?;
106// Play a sound 2 ticks (beats) from now.
107let sound_data_1 = StaticSoundData::from_file("sound1.ogg")?
108	.start_time(clock.time() + 2);
109manager.play(sound_data_1)?;
110// Play a different sound 4 ticks (beats) from now.
111let sound_data_2 = StaticSoundData::from_file("sound2.ogg")?
112	.start_time(clock.time() + 4);
113manager.play(sound_data_2)?;
114// Start the clock.
115clock.start();
116# Result::<(), Box<dyn std::error::Error>>::Ok(())
117```

119## Features
120
121The Kira crate has the following feature flags:
122
123- `cpal` (enabled by default) - enables the cpal backend and makes it the default for
124  audio managers. This allows Kira to talk to the operating system to output audio. Most
125  users should leave this enabled.
126- `symphonia` (enabled by default) - allows loading and streaming audio from common
127  audio formats, like MP3 and WAV.
128	- `mp3` (enabled by default) - enables support for loading and streaming MP3 audio (enables
129		the `symphonia` feature automatically)
130	- `ogg` (enabled by default) - enables support for loading and streaming OGG audio (enables
131		the `symphonia` feature automatically)
132	- `flac` (enabled by default) - enables support for loading and streaming FLAC audio (enables
133		the `symphonia` feature automatically)
134	- `wav` (enabled by default) - enables support for loading and streaming WAV audio (enables
135		the `symphonia` feature automatically)
136- `serde` - adds `Serialize` and `Deserialize` implementations for the following types:
137	- [`Capacities`]
138	- [`ClockSpeed`](crate::clock::ClockSpeed)
139	- [`DistortionKind`](crate::effect::distortion::DistortionKind)
140	- [`Easing`]
141	- [`EndPosition`](crate::sound::EndPosition)
142	- [`EqFilterKind`](crate::effect::eq_filter::EqFilterKind)
143	- [`FilterMode`](crate::effect::filter::FilterMode)
144	- [`Frame`]
145	- [`Mapping`]
146	- [`PlaybackPosition`](crate::sound::PlaybackPosition)
147	- [`PlaybackRate`]
148	- [`PlaybackState`](crate::sound::PlaybackState)
149	- [`Region`](crate::sound::Region)
150	- [`Decibels`]
151	- [`Waveform`](crate::modulator::lfo::Waveform)
152- `assert_no_alloc` - uses the [`assert_no_alloc`](https://crates.io/crates/assert_no_alloc) crate
153  to cause panics if memory is allocated or deallocated on the audio thread. This is mainly useful
154  for people developing Kira itself.
155
156## Loading other audio file formats
157
158Kira will be able to load any audio format that Symphonia supports with its
159current enabled features. For example, to add support for AAC files, you can
160add `symphonia` to your Cargo.toml with the `aac` feature:
161
162```toml
163symphonia = { version = "0.5.2", features = ["aac"] }
164```
165
166Kira's `mp3`, `ogg`, `flac`, and `wav` feature flags are provided for convenience.
167
168See the [symphonia documentation](https://github.com/pdeljanov/Symphonia#formats-demuxers)
169for a list of supported container formats and codecs.
170
171## Performance using the `dev` profile
172
173By default, Rust programs run with the `dev` profile are not optimized. This can
174lead to poor performance of audio playback and long loading times for audio
175files. You can alleviate this by building Kira and its audio-related
176dependencies with a higher optimization level. Add the following to your
177Cargo.toml:
178
179```toml
180[profile.dev.package.kira]
181opt-level = 3
182
183[profile.dev.package.cpal]
184opt-level = 3
185
186[profile.dev.package.symphonia]
187opt-level = 3
188
189[profile.dev.package.symphonia-bundle-mp3]
190opt-level = 3
191
192[profile.dev.package.symphonia-format-ogg]
193opt-level = 3
194
195[profile.dev.package.symphonia-codec-vorbis]
196opt-level = 3
197
198[profile.dev.package.symphonia-bundle-flac]
199opt-level = 3
200
201[profile.dev.package.symphonia-format-wav]
202opt-level = 3
203
204[profile.dev.package.symphonia-codec-pcm]
205opt-level = 3
206```
207
208You can also build all of your projects with a higher optimization level by
209using this snippet instead:
210
211```toml
212[profile.dev.package."*"]
213opt-level = 3
214```
215
216Building dependencies with a higher optimization level does increase compile
217times, but only when compiling your project from scratch. If you only make
218changes to your crate, you're not recompiling the dependencies, so you don't
219suffer from the longer compilation step in that case. Building dependencies
220optimized and the main crate unoptimized can be a good balance of performance
221and compile times for games.
